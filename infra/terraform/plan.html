⚡ Improvements & Considerations

🔒 Security

  • Azure Bastion replaces Jump VM with open RDP/SSH — no public ports, browser-based, Entra ID auth
  • AKS Workload Identity — federated service account per pod, scoped access to Key Vault only
  • Key Vault RBAC mode — use role assignments instead of legacy access policies, full audit trail
  • NSGs on every subnet — deny all internet inbound; allow only AKS subnet → data subnets
  • ACR private endpoint — AKS nodes pull images without leaving the VNet
  • Private AKS cluster — kube-apiserver has no public endpoint, reachable only via Jump VNet peering

📈 Scalability

  • AKS: 3 node pools — System (control), User (API/frontend), Transcoder (tainted for FFmpeg pods only)
  • Azure CNI Overlay — pods get real VNet IPs, required for private endpoint reachability without extra routing
  • AKS subnet /22 — 1024 IPs minimum; Azure CNI burns IPs fast, /24 will run out at scale
  • App VNet /16 — generous address space; VNet CIDR cannot be resized without recreation
  • Redis Standard SKU — has SLA and replication; Basic has no SLA and is not prod-ready
  • Cosmos Session consistency — right tradeoff for streaming metadata; choose before first write, changing is breaking

🌐 Networking (known gotchas)

  • Private DNS Zones required — without them, DNS resolves to public IP even with public access disabled
  • PostgreSQL needs a delegated subnet — Flexible Server requires Microsoft.DBforPostgreSQL/flexibleServers delegation; cannot share with other services
  • Non-overlapping CIDRs — Jump VNet and App VNet must have different address spaces for peering to work
  • Blob endpoint — use blob.core.windows.net (not DFS endpoint) for HLS serving
  • AKS private cluster API EP — Azure creates a managed VNet; plan peering before provisioning

🔭 Observability & Operations

  • Log Analytics Workspace — central sink for AKS diagnostics, Key Vault audit logs, PostgreSQL, NSG flow logs
  • Azure Monitor + Container Insights — out-of-the-box pod/node metrics, HPA integration
  • Helm before Terraform — validate all K8s manifests locally first, then wrap in Terraform AzureRM provider
  • Separate Terraform state per layer — networking, AKS, services in separate state files to avoid blast radius
  • Transcoder node pool taints — prevents API/frontend pods scheduling on expensive GPU/high-CPU nodes
